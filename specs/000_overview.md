## アプリ概要

### 目的

- 友達とMT後に行く店選びにかかる時間を減らす

### 現状

- 食べたいものを出し合う
- マップで調べる
- 営業中か調べる

### 課題

- マップアプリでは候補の比較が難しい
- 土地勘がないとマップでいちいち店を調べないと行けなくて、どんな店があるか調べるのに時間がかかる

### アプローチ

- SNSのタイムラインのように周辺の店を見られるようにする
- 営業中かどうか、レビュー、料金、位置、カテゴリなどが一目でわかるようにする
- 「検索」を「選択」に
- 気になった店をマークし、マークされた店だけを比較できるようにする

- データソース
  - 店舗情報は Google Maps Platform（Places API の Place 形式）を基準に扱う

### 対象のユーザ
- 日本の理系大学生
- スマホでの使用をメインに想定

### ポイント

- マップとの差別化
    - 極限まで「検索」を減らす。選択で代用する
    - 複数の候補の「比較」の容易化
- スワイプとタップ操作の楽しさ
